== Overview ==

Centralized dependency version specifications table auto-generated from setup.py, providing version constraints for all transformers dependencies.

=== Description ===

The `dependency_versions_table` module contains a single `deps` dictionary mapping package names to their version specifiers. This is the source of truth for dependency versions used by `dependency_versions_check.py` and other utilities. The file is auto-generated by running `make deps_table_update` which extracts specs from setup.py.

=== Usage ===

Access the `deps` dictionary to look up version requirements for any transformers dependency. Used internally by version checking and documentation generation.

== Usage Examples ==

=== Looking Up Version Requirements ===
<syntaxhighlight lang="python">
from transformers.dependency_versions_table import deps

# Check torch version requirement
print(f"Required torch: {deps['torch']}")  # "torch>=2.2"

# Check tokenizers compatibility
print(f"Required tokenizers: {deps['tokenizers']}")  # "tokenizers>=0.22.0,<=0.23.0"

# List all tracked dependencies
print(f"Total dependencies tracked: {len(deps)}")
</syntaxhighlight>

=== Programmatic Version Checking ===
<syntaxhighlight lang="python">
from packaging import version
from transformers.dependency_versions_table import deps
import torch

# Parse and check torch version
req = deps["torch"]  # "torch>=2.2"
min_version = req.split(">=")[1].split(",")[0]
installed = torch.__version__

if version.parse(installed) >= version.parse(min_version):
    print(f"torch {installed} meets requirement {req}")
</syntaxhighlight>
